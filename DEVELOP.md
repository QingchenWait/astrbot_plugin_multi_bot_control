# 开发记录

## v0.2.2

发布日期：2026-06-13

### 修复

- 修复机器人名单填写后仍可能不生效的问题。
- 机器人身份识别改为多路径提取 QQ 号：`event.get_sender_id()`、`message_obj.sender.user_id`、OneBot 原始 payload 的 `user_id` / `sender.user_id`。
- 当前受控机器人识别改为多路径提取 QQ 号：`event.get_self_id()`、`message_obj.self_id`、OneBot 原始 payload 的 `self_id`。
- 显式区分 OneBot QQ 号与 QQ 官方 API openid。QQ 官方 API 无法用 QQ 号精确匹配时不再误判。
- 实现 `targeting.require_explicit_target`：开启后，机器人消息必须 @ 当前机器人、引用回复当前机器人或符合配置的目标规则才会被放行。
- 修复本群 `/mbot add` 误添加受控机器人 QQ 时覆盖全局受控名单的问题。全局受控名单优先。
- 修复 `AtAll` 兼容判断，避免旧环境没有 `AtAll` 类时把任意 `At` 误判为 @全体。
- 会话统计 key 改为 `平台实例 + 群聊 + 当前受控机器人 + 对端机器人`，避免不同群聊或不同平台实例互相影响。
- 人类消息只重置当前群的“无人类介入”统计，不再清空所有同群单对机器人冷却。
- LLM 请求阶段兜底拦截逻辑与前置路由使用同一套名单识别规则，减少漏拦截。
- 修复计数时机错误：收到其他 bot 的调用请求时只做预检查和挂起标记，必须等 AstrBot `after_message_sent` 钩子确认当前受控机器人已经成功发送回复后，才增加轮次、群窗口、无人类连续对话和重复消息统计。
- 修复强制唤醒错误：插件只控制 AstrBot 原本已经唤醒的机器人消息，不再把未命中原始唤醒规则的 bot 消息改成 `is_wake` / `is_at_or_wake_command`。

### 性能与资源

- 群窗口时间序列从普通 list 改为 `deque`，过期数据从头部弹出，降低长期运行成本。
- 单会话重复消息记录使用固定长度 `deque(maxlen=32)`。
- 运行态群窗口和会话状态增加上限，避免轻量服务器长时间运行时内存无界增长。
- 常用身份匹配通过一次性构造 QQ 索引完成，避免对同一名单反复线性扫描。

### 文档

- 重写 `README.md`，说明适用平台、名单配置、跨群隔离和排障步骤。
- 新增 `DEVELOP.md`，记录版本变动。
- 新增 `STRUCTURE.md`，记录文件结构、内部 API 和配置映射。
- 更新 `_conf_schema.json` 的提示语，与 v0.2.2 行为保持一致。

### 测试

- 新增 `tests/test_core.py`，使用轻量 AstrBot stub 覆盖：
  - 全局 QQ 名单识别
  - 本群名单不覆盖全局受控名单
  - 必须明确目标时的静默拦截
  - 仅收到 bot 请求但未实际发送回复时不计入轮次
  - 平台发送失败时不提交挂起轮次
  - 未被 AstrBot 原始唤醒规则唤醒的 bot 消息不会被插件强制唤醒
  - 同一对机器人跨群独立统计
  - QQ 官方 openid 不误匹配为 QQ 号

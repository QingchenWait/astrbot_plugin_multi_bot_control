# Astrbot 群聊多机器人共存控制

群里有多个 bot？使用黑名单太简单粗暴？
本插件可实现灵活的 AstrBot 的群聊多机器人共存控制，灵活、全面地让群里的机器人在多种控制策略下，进行完全可控的交流。
既不担心无限循环烧光 token，也能让机器人之间互动起来 ~

- 可登记群内所有机器人信息 (包括 AstrBot 自己创建的机器人，和来自其他渠道的机器人)。
- 支持精细的目标、轮次、冷却、频率、重复消息等多重交互控制策略。
- 可通过 WebUI / 群主命令 修改机器人名单。
- 可自定义机器人之间互相交互时的提示词，支持角色识别。


## WebUI 配置

WebUI 插件配置里可以设置：

- 全局受控机器人名单 (AstrBot 自己创建的机器人)，需要通过 QQ 号精确匹配。
- 全局不受控机器人名单 (来自外部渠道的机器人)，需要通过 QQ 号精确匹配。
- 是否要求机器人消息明确指向当前机器人。
- 策略配置：单机器人会话轮次、达到限制后的冷却、机器人请求前等待、群级时间窗口、无人类介入统计窗口和冷却、重复消息等。
- 机器人之间交流的提示词模板，支持 `<botname>`、`<callname>`、`<botid>`、`<selfname>`、`<sendername>`。

注意：受控机器人，存在“优先级”的概念。这是由插件根据机器人身份，自动生成的伪随机数，保存在 AstrBot 的 `data/plugin_data/astrbot_plugin_multi_bot_control/controlled_priorities.json`。这样，多个使用相同受控机器人名单的 AstrBot 实例，就会按照一致的顺序轮流发言，避免请求拥塞。

## 群主命令

QQ 群群主可以通过命令设置当前群的本地机器人名单。本地名单只在当前群生效，不会修改 WebUI 全局配置，也不能修改交互限制。

```text
/mbot add <QQ号> [昵称] [称呼]
/mbot remove <QQ号>
/mbot list
/mbot clear
/mbot help
```

默认本地添加的机器人会作为“不受控机器人”识别。
机器人识别仅支持 QQ 号精确匹配，不再支持按昵称或别名识别。
配置中的昵称、称呼和平台事件中的发送者昵称只会作为提示词模板通配符内容，不参与机器人身份识别。
